Оплата вручную — модуль пополнения баланса

Описание
Модуль добавляет способ «Оплата вручную» в меню пополнения баланса. Пользователь выбирает сумму, получает реквизиты (карта/СБП/реквизиты/ссылка) и отправляет скриншот оплаты администратору для подтверждения. Интеграция с меню выполняется через хук providers_config (новая система платежей).

Возможности
- Кнопка «Оплата вручную» в меню «Пополнить» через providers_config
- Выбор фиксированной суммы или ввод произвольной суммы
- Отправка реквизитов (карта, СБП, реквизиты, ссылка)
- Отправка скриншота для модерации администратору
- Deep-link подтверждение/отклонение админом (при отправке в топик)
- Поддержка fast-flow (недостающая сумма): показ суммы и инструкций для ручной оплаты

Установка и включение
1) Положите модуль в каталог modules/manual_pay
2) Убедитесь, что ядро загружает модульные роутеры и вебхуки:
   - utils/modules_loader.load_modules_from_folder — для кнопок и хендлеров
   - utils/modules_loader.load_module_webhooks — не требуется (webhook отсутствует)
   - web.register_web_routes — общий регистратор модульных вебхуков (для других модулей)
3) Проверьте настройки в modules/manual_pay/settings.py

Настройки (modules/manual_pay/settings.py)
- ENABLE — включает/выключает модуль
- MODE — режим выдачи реквизитов: "card" | "sbp" | "requisites" | "link"
- MULTI_METHODS — список типов для одновременной выдачи (например, ["card", "requisites"])
- CARDS — список номеров карт для MODE="card"
- SBP — список словарей {phone, bank} для MODE="sbp"
- REQUISITES — многострочные реквизиты для MODE="requisites"
- LINKS — список ссылок для MODE="link"
- TARGET — "admins" (рассылка всем админам) или "topic" (пост в топик супергруппы)
- CHAT_ID, THREAD_ID — параметры для TARGET="topic"
- SIGN_KEY — ключ подписи для deep-link подтверждения/отклонения (TARGET="topic")
- TEXT_* — дополнительные тексты

Интеграция с меню (providers_config)
Модуль экспортирует провайдера:
{
  "MANUAL": {
    "currency": "RUB",
    "value": "manualpay|start",
    "fast": null,
    "module": "manual_pay"
  }
}

Быстрый флоу (fast flow)
- Fast-flow для ручной оплаты не используется (требуется модерация админом), поэтому параметр fast = null.

Callbacks
- manualpay|start — старт выбора суммы
- manualpay|amount|{N}|custom — выбор фиксированной суммы / ввод произвольной
- manualpay|confirm — получить реквизиты и кнопки «Я оплатил/Отмена»
- manualpay|paid — перейти к отправке скриншота
- manualpay|send — отправить скриншот на проверку админам/в топик
- manualpay|approve|{user_id}|{amount} — админ подтвердил (зачисление)
- manualpay|reject|{user_id}|{amount} — админ отклонил

Зависимости
- aiogram (ядро бота)
- SQLAlchemy (ядро бота)
